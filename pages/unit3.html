หน่วยที่ 3

การคิดเชิงคำนวณและตรรกะของ AI

เรียนรู้วิธีคิดแบบนักวิทยาการคอมพิวเตอร์ ทำความเข้าใจอัลกอริทึม Machine Learning Neural Network และกระบวนการฝึก AI ให้ฉลาด

3.1 Computational Thinking (การคิดเชิงคำนวณ)

Computational Thinking (การคิดเชิงคำนวณ) คือ กระบวนการคิดอย่างเป็นระบบเพื่อแก้ปัญหาที่ซับซ้อน โดยใช้แนวคิดเดียวกับที่นักวิทยาการคอมพิวเตอร์ใช้ในการออกแบบโปรแกรมและระบบ AI ไม่ใช่แค่การเขียนโค้ด แต่เป็น วิธีคิด ที่ทุกคนสามารถนำไปใช้ในชีวิตประจำวันได้

การคิดเชิงคำนวณประกอบด้วย 4 ขั้นตอนหลัก:

ขั้นที่ 1: Decomposition (การแบ่งย่อยปัญหา)

คือการนำปัญหาใหญ่ที่ซับซ้อนมาแบ่งออกเป็นปัญหาย่อยๆ ที่จัดการได้ง่ายขึ้น เหมือนกับการตัดเค้กเป็นชิ้นเล็กๆ เพื่อให้กินได้สะดวก

🏠 ตัวอย่างในชีวิตจริง: การทำความสะอาดบ้าน

ปัญหาใหญ่: "ทำความสะอาดบ้านทั้งหลัง" → แบ่งย่อยเป็น:

  • กวาดพื้นห้องนอน
  • ถูพื้นห้องนั่งเล่น
  • ล้างจานในครัว
  • เช็ดกระจกห้องน้ำ
  • จัดของบนโต๊ะทำงาน

🤖 ตัวอย่างใน AI: การสร้างระบบแนะนำภาพยนตร์ → แบ่งเป็น: เก็บข้อมูลผู้ใช้, วิเคราะห์รสนิยม, จัดหมวดหมู่หนัง, คำนวณความคล้าย, แสดงผลแนะนำ

ขั้นที่ 2: Pattern Recognition (การจดจำรูปแบบ)

คือการสังเกตหา "รูปแบบ" หรือ "ความคล้ายคลึง" ในปัญหาย่อยต่างๆ ถ้าเราเจอรูปแบบที่ซ้ำกัน เราสามารถใช้วิธีแก้ปัญหาเดียวกันได้ ทำให้ประหยัดเวลาและทรัพยากร

🌤️ ตัวอย่างในชีวิตจริง: การพยากรณ์อากาศ

สังเกตรูปแบบ: "ทุกครั้งที่เมฆมืดครึ้ม + ลมพัดแรง + อากาศอบอ้าว = มักจะมีฝนตก" → เราจำรูปแบบนี้ได้และทำนายได้ว่าจะต้องพกร่มวันนี้

🤖 ตัวอย่างใน AI: AI วิเคราะห์อีเมลนับล้าน พบรูปแบบว่าอีเมลสแปมมักมี: หัวข้อตัวพิมพ์ใหญ่ทั้งหมด, มีลิงก์แปลก, ใช้คำว่า "ฟรี" หรือ "รางวัล" บ่อย → ใช้รูปแบบนี้กรองสแปม

ขั้นที่ 3: Abstraction (การคิดเชิงนามธรรม)

คือการตัดรายละเอียดที่ไม่จำเป็นออก เหลือเฉพาะสิ่งที่สำคัญ เพื่อให้เห็นภาพรวมของปัญหาชัดเจนขึ้น เหมือนการดูแผนที่ ที่แสดงเฉพาะถนนหลักโดยไม่แสดงต้นไม้ทุกต้น

🗺️ ตัวอย่างในชีวิตจริง: แผนที่รถไฟฟ้า

แผนที่รถไฟฟ้า BTS/MRT ไม่ได้แสดงระยะทางจริง ไม่มีตึก ไม่มีถนน มีเพียง "สถานี" และ "เส้นทาง" → นี่คือ Abstraction ที่เอาเฉพาะข้อมูลที่จำเป็นสำหรับการเดินทาง

🤖 ตัวอย่างใน AI: ในการจำแนกแมวกับสุนัข AI ไม่สนใจสีพื้นหลัง แสงแดด หรือมุมกล้อง (ตัดออก) แต่สนใจเฉพาะ: รูปร่างหู, ความยาวจมูก, รูปร่างลำตัว (เก็บไว้)

ขั้นที่ 4: Algorithm Design (การออกแบบอัลกอริทึม)

คือการสร้างชุดขั้นตอนที่ชัดเจนและเป็นลำดับ เพื่อแก้ปัญหานั้นอย่างเป็นระบบ สามารถทำซ้ำได้ทุกครั้งแล้วได้ผลเหมือนเดิม เหมือนสูตรอาหารที่ใครทำตามก็ได้ผลลัพธ์เดียวกัน

🍳 ตัวอย่างในชีวิตจริง: สูตรทำไข่เจียว

  1. ตอกไข่ 2 ฟอง ใส่ชาม
  2. เติมน้ำปลา 1 ช้อนชา
  3. ตีไข่ให้เข้ากัน
  4. ตั้งกระทะ ใส่น้ำมัน รอให้ร้อน
  5. เทไข่ลงกระทะ
  6. รอจนไข่สุกด้านล่าง แล้วพลิก
  7. ตักใส่จาน เสิร์ฟ

🤖 ตัวอย่างใน AI: อัลกอริทึมการค้นหาเส้นทางใน Google Maps: 1) รับจุดเริ่มต้นและปลายทาง 2) หาเส้นทางทั้งหมดที่เป็นไปได้ 3) คำนวณระยะทาง/เวลาแต่ละเส้นทาง 4) เลือกเส้นทางที่เร็วที่สุด 5) แสดงผลบนแผนที่

3.2 อัลกอริทึม (Algorithm)

อัลกอริทึม (Algorithm) คือ ชุดคำสั่งหรือขั้นตอนที่ชัดเจน มีลำดับแน่นอน สำหรับแก้ปัญหาหรือทำงานให้สำเร็จ เปรียบเสมือน "สูตรอาหาร" ของคอมพิวเตอร์ ที่บอกทีละขั้นตอนว่าต้องทำอะไรบ้างเพื่อให้ได้ผลลัพธ์ที่ต้องการ

อัลกอริทึมเป็นหัวใจสำคัญของ AI เพราะ AI ทำงานตามอัลกอริทึมที่มนุษย์ออกแบบไว้ทั้งหมด

คุณสมบัติ 5 ประการ ของอัลกอริทึมที่ดี

1. ความชัดเจน

แต่ละขั้นตอนต้องชัดเจน ไม่คลุมเครือ ใครอ่านก็เข้าใจตรงกัน

2. มี Input

ต้องรับข้อมูลนำเข้า อย่างน้อย 0 ค่าขึ้นไป

3. มี Output

ต้องให้ผลลัพธ์ออกมา อย่างน้อย 1 ค่า

4. มีจุดสิ้นสุด

ต้องหยุดทำงานได้ ไม่ทำงานวนไปเรื่อยๆ ไม่มีวันจบ

5. มีประสิทธิภาพ

ทำงานเสร็จภายในเวลาที่เหมาะสม ไม่ช้าเกินไป

ตัวอย่าง: Bubble Sort (การเรียงลำดับแบบฟองสบู่)

Bubble Sort เป็นอัลกอริทึมการเรียงลำดับที่เข้าใจง่ายที่สุด ทำงานโดยการเปรียบเทียบข้อมูลทีละคู่ที่อยู่ติดกัน แล้วสลับที่ถ้าเรียงผิดลำดับ ทำซ้ำไปเรื่อยๆ จนข้อมูลเรียงเสร็จ

📝 ขั้นตอนของ Bubble Sort:

  1. เริ่มจากข้อมูลตัวแรก เปรียบเทียบกับตัวถัดไป
  2. ถ้าตัวแรกมากกว่าตัวถัดไป → สลับที่กัน
  3. เลื่อนไปเปรียบเทียบคู่ถัดไป ทำจนครบทุกคู่
  4. เมื่อผ่านรอบแรก ตัวที่มากที่สุดจะ "ลอย" ไปอยู่ท้ายสุด (เหมือนฟองสบู่ลอยขึ้น)
  5. ทำซ้ำจนกว่าจะไม่มีการสลับที่เลย = เรียงเสร็จ

ตัวอย่าง: Binary Search (การค้นหาแบบทวิภาค)

Binary Search เป็นอัลกอริทึมการค้นหาที่เร็วมาก ใช้หลักการ "แบ่งครึ่ง" ข้อมูลที่เรียงแล้ว ทุกครั้งที่เปรียบเทียบจะตัดข้อมูลออกครึ่งหนึ่ง ทำให้หาคำตอบได้เร็วมาก

📖 เปรียบเทียบกับชีวิตจริง: การเปิดพจนานุกรม

ถ้าเราต้องการหาคำว่า "mango" ในพจนานุกรม:

  • เปิดตรงกลางเล่ม → เจอตัว "L" → "M" อยู่หลังจากนี้ → ตัดครึ่งแรกออก
  • เปิดตรงกลางครึ่งหลัง → เจอตัว "P" → "M" อยู่ก่อนหน้า → ตัดครึ่งหลังออก
  • เปิดตรงกลางส่วนที่เหลือ → เจอตัว "M" → ค้นหาต่อจนเจอ "mango"!

⚡ ประสิทธิภาพ: ในข้อมูล 1,000,000 ตัว ค้นหาแบบธรรมดาอาจต้องเทียบ 1,000,000 ครั้ง แต่ Binary Search ใช้เพียง ~20 ครั้ง!

3.3 Machine Learning (การเรียนรู้ของเครื่อง)

Machine Learning (ML) คือ สาขาย่อยของ AI ที่ทำให้คอมพิวเตอร์สามารถ "เรียนรู้" จากข้อมูลและปรับปรุงตัวเองได้ โดยไม่ต้องเขียนโปรแกรมบอกทุกขั้นตอนอย่างชัดเจน แตกต่างจากการเขียนโปรแกรมแบบดั้งเดิมที่ต้องบอกคอมพิวเตอร์ทุกอย่างว่าต้องทำอะไร

เปรียบเทียบ: การเขียนโปรแกรมแบบดั้งเดิม vs Machine Learning

โปรแกรมแบบดั้งเดิม

กฎ + ข้อมูล → ผลลัพธ์

มนุษย์เขียนกฎเองทุกข้อ คอมพิวเตอร์ทำตามกฎเท่านั้น

  • ต้องรู้คำตอบล่วงหน้า
  • เขียนเงื่อนไข if-else ทุกกรณี
  • แก้ไขยากเมื่อมีกรณีใหม่
  • เหมาะกับปัญหาที่ชัดเจน

ตัวอย่าง: "ถ้าอุณหภูมิ > 38°C → แจ้งว่ามีไข้"

Machine Learning

ข้อมูล + ผลลัพธ์ → กฎ

ป้อนข้อมูลพร้อมคำตอบ ให้ ML หากฎเอง

  • เรียนรู้จากตัวอย่างได้เอง
  • จัดการกับข้อมูลซับซ้อนได้
  • ปรับตัวเมื่อมีข้อมูลใหม่
  • เหมาะกับปัญหาที่ซับซ้อน

ตัวอย่าง: "ให้ข้อมูลคนไข้ 10,000 คน ML จะหาเองว่าอาการแบบไหนเป็นโรคอะไร"

Machine Learning 3 ประเภท

1. Supervised Learning

การเรียนรู้แบบมีผู้สอน

ป้อนข้อมูลพร้อม "คำตอบที่ถูกต้อง" (Label) ให้ AI เรียนรู้ เหมือนครูให้โจทย์พร้อมเฉลย AI จะเรียนรู้ความสัมพันธ์ระหว่างข้อมูลกับคำตอบ แล้วนำไปทำนายข้อมูลใหม่ได้

ตัวอย่างการใช้งาน:

  • กรองอีเมลสแปม - สอนว่าอีเมลไหนเป็นสแปม/ไม่ใช่
  • ทำนายราคาบ้าน - ให้ข้อมูลบ้าน+ราคาจริง แล้วทำนายราคาบ้านใหม่
  • วินิจฉัยโรค - ให้ภาพ X-ray + ผลวินิจฉัย AI เรียนรู้จำแนกโรค
  • จดจำใบหน้า - สอนว่ารูปนี้เป็นใคร

2. Unsupervised Learning

การเรียนรู้แบบไม่มีผู้สอน

ป้อนข้อมูลโดยไม่บอกคำตอบ ให้ AI ค้นหา "รูปแบบ" และจัดกลุ่มข้อมูลเอง เหมือนให้เด็กจัดของเล่นเป็นกลุ่มโดยไม่บอกว่าต้องจัดอย่างไร เด็กจะหาวิธีจัดเอง

ตัวอย่างการใช้งาน:

  • จัดกลุ่มลูกค้า - แบ่งลูกค้าตามพฤติกรรมการซื้อ โดยไม่ต้องกำหนดกลุ่มล่วงหน้า
  • หาความผิดปกติ - ตรวจจับธุรกรรมทางการเงินที่ผิดปกติ (Fraud Detection)
  • แนะนำสินค้า - จัดกลุ่มสินค้าที่มักถูกซื้อด้วยกัน
  • บีบอัดข้อมูล - หาข้อมูลที่ซ้ำซ้อนและลดขนาด

3. Reinforcement Learning

การเรียนรู้แบบเสริมแรง

AI เรียนรู้จากการ "ลองทำ" ถ้าทำถูกได้รางวัล (+reward) ถ้าทำผิดถูกลงโทษ (-penalty) ค่อยๆ ปรับพฤติกรรมจนเก่งขึ้น เหมือนฝึกสุนัขด้วยขนม

ตัวอย่างการใช้งาน:

  • AlphaGo - เรียนรู้เล่นเกมโกะจนชนะแชมป์โลก
  • รถยนต์ไร้คนขับ - เรียนรู้ขับรถจากการจำลอง ได้รางวัลเมื่อขับปลอดภัย
  • หุ่นยนต์ - เรียนรู้เดิน จับของ จากการลองผิดลองถูก
  • ChatGPT (RLHF) - ปรับปรุงคำตอบจาก feedback ของมนุษย์

3.4 Neural Network (โครงข่ายประสาทเทียม)

Neural Network (โครงข่ายประสาทเทียม) คือ โมเดลทางคณิตศาสตร์ที่ได้แรงบันดาลใจจากการทำงานของสมองมนุษย์ ประกอบด้วย "เซลล์ประสาทเทียม" (Neurons) จำนวนมากที่เชื่อมต่อกันเป็นชั้นๆ (Layers) เพื่อประมวลผลข้อมูลที่ซับซ้อน

โครงสร้าง 3 ชั้นหลัก

Input Layer

ชั้นรับข้อมูลเข้า เช่น ค่าสีของแต่ละพิกเซลในรูปภาพ ค่าความถี่ของเสียง หรือตัวเลขต่างๆ แต่ละ neuron รับข้อมูลเข้า 1 ค่า

Hidden Layer(s)

ชั้นประมวลผลที่ซ่อนอยู่ ทำหน้าที่วิเคราะห์ หาความสัมพันธ์ และแปลงข้อมูล ยิ่งมีหลายชั้น (Deep) ยิ่งเรียนรู้สิ่งซับซ้อนได้

Output Layer

ชั้นแสดงผลลัพธ์ เช่น ผลการจำแนก (แมว/สุนัข) ค่าทำนาย (ราคา) หรือข้อความที่สร้างขึ้น แต่ละ neuron คือ 1 คำตอบที่เป็นไปได้

Deep Learning คืออะไร?

Deep Learning (การเรียนรู้เชิงลึก) คือ Neural Network ที่มี Hidden Layer จำนวนมาก (ตั้งแต่ 3 ชั้นขึ้นไป จนถึงหลายร้อยชั้น) คำว่า "Deep" มาจากความ "ลึก" ของจำนวนชั้น ทำให้สามารถเรียนรู้รูปแบบที่ซับซ้อนมากๆ ได้

🧠 เปรียบเทียบกับสมองมนุษย์:

สมองมนุษย์:

  • เซลล์ประสาท ~86,000 ล้านเซลล์
  • จุดเชื่อมต่อ ~100 ล้านล้านจุด
  • ใช้พลังงานเพียง ~20 วัตต์
  • เรียนรู้ได้ตลอดชีวิต

GPT-4 (ใหญ่ที่สุด 2024):

  • พารามิเตอร์ ~1.76 ล้านล้านค่า
  • ใช้พลังงาน ~หลายเมกะวัตต์
  • เรียนรู้จากข้อมูลบนอินเทอร์เน็ต
  • ฝึกครั้งเดียว ไม่เรียนรู้เพิ่มเอง

💡 แม้ AI จะใหญ่มาก แต่ยังซับซ้อนน้อยกว่าสมองมนุษย์มาก และไม่มีจิตสำนึก ไม่ "เข้าใจ" สิ่งที่ทำจริงๆ

3.5 การฝึกโมเดล AI (AI Training Pipeline)

การสร้าง AI ที่ใช้งานได้จริง ไม่ใช่แค่เขียนโค้ดแล้วจบ แต่ต้องผ่านกระบวนการฝึกฝน (Training) ที่เป็นระบบ เหมือนการฝึกนักกีฬา ที่ต้องมีโค้ช มีแผนฝึก มีการทดสอบ กว่าจะแข่งขันได้ AI ก็ต้องผ่าน 5 ขั้นตอนสำคัญ

ขั้นที่ 1

เตรียมข้อมูล (Data Preparation)

รวบรวมข้อมูลที่เกี่ยวข้อง ทำความสะอาดข้อมูล (ลบข้อมูลผิดพลาด ข้อมูลซ้ำ ข้อมูลไม่ครบ) แบ่งข้อมูลเป็น 3 ส่วน: Training Set (70-80%) สำหรับฝึก, Validation Set (10-15%) สำหรับปรับจูน, Test Set (10-15%) สำหรับทดสอบสุดท้าย

⚠️ ข้อมูลคุณภาพดี = AI คุณภาพดี (Garbage In, Garbage Out)

ขั้นที่ 2

เลือกโมเดล (Model Selection)

เลือกสถาปัตยกรรม AI ที่เหมาะกับงาน เช่น CNN สำหรับรูปภาพ, RNN/Transformer สำหรับภาษา, Random Forest สำหรับข้อมูลตาราง ต้องพิจารณาความซับซ้อนของปัญหา ปริมาณข้อมูล และทรัพยากรที่มี

ขั้นที่ 3

ฝึกสอน (Training)

ป้อนข้อมูล Training Set ให้โมเดล ให้โมเดลทำนาย แล้วเปรียบเทียบกับคำตอบจริง คำนวณค่าผิดพลาด (Loss) แล้วปรับค่าน้ำหนัก (Weights) ทำซ้ำหลายรอบ (Epochs) จนค่าผิดพลาดลดลงต่ำที่สุด อาจใช้เวลาหลายชั่วโมงถึงหลายสัปดาห์

ขั้นที่ 4

ทดสอบ (Testing & Evaluation)

ทดสอบโมเดลด้วย Test Set ที่ไม่เคยเห็นมาก่อน เพื่อดูว่าโมเดลทำงานได้ดีแค่ไหนกับข้อมูลใหม่ วัดค่าความแม่นยำ (Accuracy) ค่า Precision, Recall ถ้าผลไม่ดีพอ ต้องกลับไปปรับขั้นตอนก่อนหน้า

ขั้นที่ 5

นำไปใช้งานจริง (Deployment)

นำโมเดลที่ผ่านการทดสอบแล้วไปติดตั้งในระบบจริง เช่น ทำเป็น API, ใส่ในแอป, หรือฝังในอุปกรณ์ ต้องมีการตรวจสอบประสิทธิภาพอย่างต่อเนื่อง และปรับปรุงเมื่อมีข้อมูลใหม่ (MLOps)

3.6 Overfitting vs Underfitting

ในการฝึก AI มีปัญหาสำคัญ 2 อย่างที่พบบ่อย คือ Overfitting (เรียนมากเกินไป) และ Underfitting (เรียนน้อยเกินไป) ทั้งสองอย่างทำให้ AI ทำงานได้ไม่ดี เปรียบเทียบง่ายๆ กับนักเรียนที่เตรียมสอบ:

🔴 Overfitting (เรียนมากเกินไป)

AI จำข้อมูลฝึกได้หมดทุกรายละเอียด รวมทั้ง "สัญญาณรบกวน" (noise) ทำให้ทำงานดีมากกับข้อมูลฝึก แต่ล้มเหลวเมื่อเจอข้อมูลใหม่

📚 เปรียบเทียบ: นักเรียนที่ท่องจำ

เหมือนนักเรียนที่ท่องจำข้อสอบเก่าทุกข้อ ทุกตัวเลือก โดยไม่เข้าใจหลักการ → เมื่อเจอข้อสอบใหม่ที่โจทย์ต่างไปนิดเดียว ก็ตอบไม่ได้เลย

🔧 วิธีแก้:

  • เพิ่มข้อมูลฝึกให้หลากหลายขึ้น
  • ลดความซับซ้อนของโมเดล
  • ใช้เทคนิค Regularization
  • ใช้ Dropout (สุ่มปิด neuron บางตัว)
  • หยุดฝึกเมื่อเหมาะสม (Early Stopping)

🟡 Underfitting (เรียนน้อยเกินไป)

AI เรียนรู้ไม่เพียงพอ ไม่สามารถจับรูปแบบที่แท้จริงในข้อมูลได้ ทำให้ทำงานได้ไม่ดีทั้งข้อมูลฝึกและข้อมูลใหม่

📚 เปรียบเทียบ: นักเรียนที่ไม่อ่านหนังสือ

เหมือนนักเรียนที่อ่านหนังสือแค่ผ่านๆ ไม่ได้ทำความเข้าใจ → ตอบไม่ได้ทั้งข้อสอบเก่าและข้อสอบใหม่ เพราะไม่เข้าใจเนื้อหาเลย

🔧 วิธีแก้:

  • เพิ่มความซับซ้อนของโมเดล (เพิ่ม layers/neurons)
  • ฝึกนานขึ้น (เพิ่ม epochs)
  • เพิ่ม features ที่เกี่ยวข้อง
  • ลด regularization ลง
  • เปลี่ยนไปใช้โมเดลที่เหมาะสมกว่า

🟢 Good Fit (พอดี) - เป้าหมายที่ต้องการ

AI เรียนรู้ "หลักการ" ที่แท้จริงจากข้อมูล ไม่จำมากเกินไป ไม่น้อยเกินไป → ทำงานได้ดีทั้งข้อมูลฝึกและข้อมูลใหม่ เหมือนนักเรียนที่เข้าใจหลักการ จึงทำข้อสอบได้ทุกรูปแบบ ไม่ว่าโจทย์จะเปลี่ยนอย่างไร

สรุปความรู้ประจำหน่วย

สิ่งที่ควรจำ

  • Computational Thinking มี 4 ขั้นตอน: Decomposition, Pattern Recognition, Abstraction, Algorithm Design
  • อัลกอริทึมที่ดีต้อง: ชัดเจน, มี Input/Output, มีจุดสิ้นสุด, มีประสิทธิภาพ
  • Machine Learning มี 3 ประเภท: Supervised, Unsupervised, Reinforcement Learning
  • Neural Network ประกอบด้วย: Input Layer, Hidden Layer(s), Output Layer
  • Deep Learning = Neural Network ที่มีหลายชั้น (Deep = ลึก)
  • การฝึก AI มี 5 ขั้นตอน: เตรียมข้อมูล → เลือกโมเดล → ฝึก → ทดสอบ → ใช้งานจริง

ประเด็นสำคัญ

  • Computational Thinking ไม่ใช่แค่เรื่องคอมพิวเตอร์ แต่เป็นทักษะชีวิตที่ใช้ได้ทุกสถานการณ์
  • ML ต่างจากโปรแกรมทั่วไป ตรงที่ "เรียนรู้กฎจากข้อมูล" แทนที่จะ "ถูกบอกกฎ"
  • ข้อมูลคือหัวใจของ AI - ข้อมูลดี AI เก่ง, ข้อมูลแย่ AI แย่
  • Overfitting = จำมากเกินไป (ท่องจำ), Underfitting = เรียนน้อยเกินไป (ไม่เข้าใจ)
  • AI ไม่ได้ "คิดเป็น" แต่ใช้คณิตศาสตร์หารูปแบบจากข้อมูล
  • GPT-4 มีพารามิเตอร์ล้านล้าน แต่ยังซับซ้อนน้อยกว่าสมองมนุษย์

คำถามชวนคิด: ลองใช้ Computational Thinking กับปัญหาในชีวิตประจำวัน เช่น "จะเตรียมตัวสอบอย่างไรให้ได้คะแนนดี?" → แบ่งย่อยเป็นวิชา → หารูปแบบข้อสอบ → สรุปเฉพาะหัวข้อสำคัญ → วางแผนอ่านหนังสือ นี่คือ Computational Thinking!
